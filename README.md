O arquivo principal do GitHub deve conter:
1.
Arquitetura: Um pequeno diagrama ou descrição do fluxo (Fonte -> Python -> Parquet -> Postgres).
2.
Documentação da tarefa: prints e descrição de cada uma das etapas, dos scripts gerados.
3.
Dicionário de Dados: Descrição de cada coluna da base escolhida.
4.
Qualidade de Dados: Relatar quais problemas foram encontrados (ex: "A coluna X tinha 20% de nulos").
5.
Instruções de Execução: Como instalar as dependências (pip install -r requirements.txt) e a ordem de execução dos scripts.
